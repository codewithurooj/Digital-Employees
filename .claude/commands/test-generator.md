# Test Generator Skill

Automatically generate pytest tests for AI Employee components.

## Purpose

This skill analyzes Python files and generates comprehensive test suites following best practices.

## Usage

```
/test-generator <file_path_or_module>
```

**Examples:**
```
/test-generator src/watchers/filesystem_watcher.py
/test-generator src/watchers/base_watcher.py
/test-generator all-watchers
```

## Instructions

### Step 1: Analyze Target

1. Read the target file specified by the user
2. Identify:
   - Classes and their methods
   - Functions
   - Dependencies and imports
   - Edge cases from docstrings/comments
   - Error handling paths

### Step 2: Generate Test Structure

Create test file at `tests/test_<module_name>.py` with:

### Step 3: Generate Specific Tests

For each class/function, generate tests covering:

#### A. Happy Path Tests
- Normal operation with valid inputs
- Expected outputs verified

#### B. Edge Case Tests
- Empty inputs
- Boundary values
- Special characters in filenames
- Large files/data

#### C. Error Handling Tests
- Invalid inputs
- Missing files/directories
- Permission errors
- Network failures (if applicable)

#### D. State Tests
- Initialization state
- State changes after operations
- Cleanup/teardown

#### For Watchers:

    def test_create_action_file_creates_markdown(self, temp_vault):

        watcher = SomeWatcher(str(temp_vault))
        # Create test item
        result = watcher.create_action_file(item)

    def test_log_action_writes_to_daily_log(self, temp_vault):
        """log_action should append to daily JSON log."""

### Step 5: Output

1. Write the test file to `tests/test_<module_name>.py`
2. Create `tests/__init__.py` if it doesn't exist
3. Create `tests/conftest.py` with shared fixtures if needed
4. Report:

   - Number of test methods generated

## Test File Template

```python

Review and customize as needed.

Run: pytest tests/test_{module_name}.py -v
Run with coverage: pytest tests/test_{module_name}.py --cov=src/{module_path}
"""

import pytest
from pathlib import Path
from datetime import datetime
from unittest.mock import Mock, patch, MagicMock
import tempfile
import json

{imports}


# =============================================================================
# Fixtures
# =============================================================================

{fixtures}


# =============================================================================
# Tests
# =============================================================================

{test_classes}


# =============================================================================
# Run Tests
# =============================================================================

if __name__ == "__main__":
    pytest.main([__file__, "-v"])
```

## Rules

1. **Always use pytest** - No unittest
2. **Use fixtures** - For setup/teardown
3. **Use tmp_path** - For file system tests
4. **Mock external dependencies** - Network, APIs, etc.
5. **Descriptive names** - `test_<method>_<scenario>_<expected>`
6. **Docstrings** - Every test needs a docstring
7. **Arrange-Act-Assert** - Clear test structure
8. **No hardcoded paths** - Use fixtures and tmp_path
